## Issue 概要
**タイトル**: security: clone target path validation bypass allows repositories outside CM_ROOT_DIR
**URL**: https://github.com/Kewton/CommandMate/issues/392

セキュリティ脆弱性: `POST /api/repositories/clone` の `customTargetPath` 処理において、`isPathSafe()` での検証後に `path.resolve(basePath, customTargetPath)` によるカノニカル化を行わずそのまま `git clone` に渡されるため、相対パスで `CM_ROOT_DIR` 外にリポジトリをクローンできる。

## 仮説検証結果（Phase 0.5）

| # | 仮説/主張 | 判定 |
|---|----------|------|
| 1 | `targetPath = customTargetPath \|\| this.getTargetPath(repoName)` で未解決パスが代入 | Confirmed |
| 2 | `isPathSafe` が相対パスを basePath 内として評価し true を返す | Confirmed |
| 3 | 検証後に `path.resolve()` カノニカル化がない | Confirmed |
| 4 | 未解決パスがそのまま DB 永続化・spawn に使われる | Confirmed |
| 5 | `spawn('git', ['clone', '--progress', cloneUrl, targetPath])` が未解決パスを使用 | Confirmed |

全仮説 Confirmed — Issue の主張は技術的に正確。
