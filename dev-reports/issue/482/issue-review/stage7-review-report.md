## Should Fix（推奨対応）

### S7-001: 親Issue更新の残存件数が不正確の可能性

**カテゴリ**: 整合性
**場所**: 受け入れ基準 - 親Issue更新の数値

**問題**:
受け入れ基準に「親Issue #475 のR-4行の残存件数（6箇所→4箇所）を更新する」とあるが、本Issueは4箇所のマーカーを対象としている。全4箇所を解消した場合、残存は 6-4=2箇所 となるはずであり、「4箇所」は計算が合わない。

**現在のコードベースの実態**:
- `route.ts:33` - TODO 1箇所
- `opencode-config.ts:217` - TODO 1箇所
- `opencode-config.ts:284` - TODO 1箇所
- `cli-patterns.ts:27` - Issue #XXX参照 1箇所
- 合計: 4箇所（Issue本文の対象一覧と一致）

**考えられる解釈**:
- 元の6箇所のうち2箇所が既に別Issueで解消済みの場合、現在の残存が4箇所であり、本Issueで4箇所全てを解消すれば「4箇所→0箇所」が正しい
- 元の6箇所のうち本Issueで4箇所を解消する場合、「6箇所→2箇所」が正しい

**推奨対応**:
親Issue #475のR-4項目の元の6箇所の内訳を確認し、残存件数の計算を明確化する。
